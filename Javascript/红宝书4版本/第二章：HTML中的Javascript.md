## 2.1 script 元素
-  将 Javascript 插入 HTML 的主要方法是使用 script 元素
- 元素属性：
1. async: 可选。表示立即下载脚本，但不阻止页面动作，下载完才加载，会阻塞页面动作。只对外部脚本文件有效
2. defer：可选。表示立即加载脚本，但不会阻塞页面动作，到页面动作完成，才会加载脚本。只对外部文件有效
3. charset：可选。使用 src 属性 制定的代码字符集。这个属性很少使用，大多浏览器不在乎他的值
4. crossorigin：可选。配置相关请求的 CORS（Cross-origin resource sharing 跨域资源共享）。默认不实用 CORS。crossorigin = 'anonymous' 配置资源请求不必设置凭证依据。crossorigin = 'use-credentials' 设置凭证依据，意味着出站请求会包含凭证。
5. integrity：可选。允许对比接收到的资源和指定的加密签名，以验证资源的完整性。接收到到资源验证不匹配和报错。脚本不执行。想换个属性可以用于确保内容分发网络不会提供恶意内容。
6. src：可选。表示包含要执行到代码和外部文件
7. type：可选。表示代码中脚本语言到内容类型。如果值等于module，代码会被当成 ES6 模块，而且只有这个时候代码才能出现 import 和 export 关键字
- 注意：不管 async 还是 defer，异步加载文件，本来顺序就有不确定性（没有顺序依赖的脚本还好说）。所以常规做法就是把文件加载放在 body 标签底部

> ps: 浏览器兼容的问题还是让人头疼。
- 相对于行内代码推荐使用外部文件的理由：
1. 可维护性：代码不散乱分布，统一在一个文件内
2. 缓存：浏览器会根据特定的设置缓存外部链接的 js 文件，两个页面都用到一个 js 文件，文件只需要下载一次。意味着页面加载更快。
